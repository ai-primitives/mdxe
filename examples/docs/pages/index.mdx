# Creating a Single-Page Documentation Site with MDXE

This guide demonstrates how to create a minimal documentation site using MDXE and Nextra with just two files. You're currently reading the result!

## Required Files

1. `package.json` - Project configuration and dependencies
```json
{
  "name": "mdxe-docs-example",
  "private": true,
  "type": "module",
  "mdxe": {
    "layouts": "./layouts",
    "components": "./components"
  },
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "next lint",
    "format": "prettier --write ."
  }
}
```

2. `page.mdx` (this file) - Your documentation content

## Getting Started

1. Create a new directory for your documentation:
```bash
mkdir my-docs
cd my-docs
```

2. Create the two required files:
   - Copy the `package.json` content above
   - Create a `page.mdx` file (like this one)

3. Install dependencies:
```bash
pnpm install
```

4. Start the development server:
```bash
pnpm dev
```

## Features Available

- 🚀 Zero-config MDX processing
- ⚡️ NextJS integration
- 🎨 Default Tailwind Typography styling
- 🔄 File watching and hot reload
- 📝 App Router metadata support

## Writing Content

You can use all standard MDX features:

### Markdown Syntax
- **Bold text**
- *Italic text*
- Lists and nested lists
- [Links](https://mdxjs.com)
- Code blocks with syntax highlighting

### React Components
You can import and use React components directly in your MDX:

```jsx
import { Button } from './components/Button'

<Button>Click me!</Button>
```

## Dependencies

The following dependencies are automatically included:
- mdxe (local dependency)
- next
- nextra
- nextra-theme-docs
- react
- react-dom
